# Техническая спецификация модуля Fletcher  
## Раздел: База данных (Data Layer)

---

### 🧠 1. Роль Fletcher
Fletcher — центральный компонент системы, отвечающий за:
- хранение данных  
- консистентность  
- связи между сущностями  
- подготовку данных для аналитики (Derek)  

> Fletcher является единственным компонентом, работающим с базой данных (PostgreSQL).

---

### 🏗 2. Принципы проектирования
- база данных — источник истины  
- минимально необходимая структура  
- отсутствие избыточных сущностей  
- отказ от хранения JSON без необходимости  
- разделение сущностей по ответственности  

---

### 📦 3. Схема базы данных

#### 3.1 `users`
Хранит пользователей Telegram.  
**Поля:**
- id (PK)  
- tg_user_id (UNIQUE)  
- access_hash  
- trust_score  
- created_at  

**Назначение:**  
- идентификация пользователя  
- хранение технических данных Telegram  
- флаг блокировки (`trust_score = 0`)  

---

#### 3.2 `tmdb_accounts`
Хранит данные интеграции с TMDb.  
**Поля:**
- id (PK)  
- user_id (FK)  
- account_id  
- access_token (ENCRYPTED)  
- list_id  
- tmdb_status  
- last_sync  

**Назначение:**  
- хранение авторизации TMDb  
- управление списками  
- отслеживание состояния синхронизации  

---

#### 3.3 `movies`
Минимальная таблица фильмов.  
**Поля:**
- id (PK)  
- tmdb_id (UNIQUE)  

**Назначение:**  
- обеспечение целостности данных  
- нормализация связей  
- отказ от использования внешнего ID как внутреннего ключа  

---

#### 3.4 `user_ratings`
Ключевая таблица системы.  
**Поля:**
- user_id (FK)  
- movie_id (FK)  
- rating  
- rated_at  

**Ограничение:** UNIQUE (user_id, movie_id)  

**Назначение:**  
- хранение оценок пользователей  
- основной источник данных для Derek  

---

#### 3.5 `recommendations`
Хранит рассчитанные рекомендации.  
**Поля:**
- user_id (FK)  
- movie_id (FK)  
- score  
- generated_at  

**PRIMARY KEY:** (user_id, movie_id)  

**Назначение:**  
- быстрый доступ к рекомендациям  
- хранение результата batch-расчёта  

---

#### 3.6 `shown_log`
Лог показанных рекомендаций.  
**Поля:**
- user_id (FK)  
- movie_id (FK)  
- shown_at  

**Назначение:**  
- исключение повторных показов  
- управление повторным показом (cooldown)  

---

### ⚡ 4. Индексы
Обязательные индексы:
- INDEX users(tg_user_id)  
- INDEX movies(tmdb_id)  
- INDEX user_ratings(user_id)  
- INDEX user_ratings(movie_id)  
- INDEX recommendations(user_id, score DESC)  

---

### 🔐 5. Безопасность
- `access_token` хранится только в зашифрованном виде  
- расшифровка выполняется внутри Fletcher  
- Teddy получает токен только временно  

---

### 🔁 6. Поток данных
1. Telegram → Hellobot  
2. Hellobot → Redis → Fletcher  
3. Fletcher → запись в БД  
4. Fletcher → формирование задач для Teddy  
5. Teddy → запрос к TMDb  
6. Fletcher → обновление БД  
7. Derek → batch-обработка (раз в сутки)  
8. Fletcher → выдача рекомендаций  

---

### ⚠️ 7. Ограничения
- `trust_score` используется только как флаг блокировки  
- отсутствует хранение пользовательских событий (осознанное решение)  
- отсутствует хранение метаданных фильмов  
- отсутствует real-time аналитика  

---

### 🧠 8. Ключевые свойства
- простота структуры  
- предсказуемость поведения  
- минимальная связность  
- готовность к масштабированию
